IBM Test Engineer

As a processor characterization engineer working on the IBM Power-series servers I had the change to work in several areas for the 2 year that I worked at IBM.

  • Processor Failure Analysis
  • Processor Characterization
  • Measurement Integrity
  • Automation
The group was responsible for all system to tester correlation for the Power-series servers which involves validating that the tests performed on the processor during fabrication and packaging are correct at the system level. 

Processor Failure Analysis

For this role I was responsible for reproducing failures and determining the root cause for both processor from the field as well as manufacturing. Reproducing the failure range from very simply powering the system with the faulty processor and observing the failure to loading in test applications directly into memory and varying voltage, frequency and temperature to simulate the fail condition.

To accomplish many of these tests (some taking over 12 hours) I utilized the Perl Scripting language to sweep voltages and frequency as well as the loading of exercisers (application in memory) to maximize the test coverage and standardize tests for more accurate results. Some of these scripts were created by the lead engineer but I began writing my own scripts once I taught myself the language. 

 

Processor Characterization

For processor characterization I was mainly concerned with pattern matching in regards to how the processor performed at the factory versus how it performed as a fully functional system. The task involved running custom tests and automation to collect data on processor power vs. temperature, max frequency value, max voltage value and other parameters to validate performance of processor. 

 

Measurement Integrity

The job of measurement integrity revolved around ensuring the accuracy of the system power distribution and how it affected the processors. Current read-back accuracy was one of the major components where system power supplies would be loaded with calibrated equipment to ensure proper supply and current readings. The measuring line-loss for specific voltage planes on the processor was also crucial in ensuring processor voltage readings were accurate. Based on the measurements that were taken we were able to offset the voltage and current sensor readings (if needed) to minimize error.

Automation

During my time at IBM I was also involved with 2 side projects with the automation team: automatic chart generation and test suite user interface creation.

To auto generate reports on field data I created a Java class to run a SQL query gathering data for a given week. I then generated a series of charts based on team needs to show how systems in the field were operating.

I wrote a Java user interface to setup and run an automated test suite (also written in Java) to replace the older test execution method involving writing complicated XML's that were then passed to the Java Ant for task generation.
